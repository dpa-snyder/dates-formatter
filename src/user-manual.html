User Manual

Date Formatter

Standardize date columns in archival Excel and CSV spreadsheets.

Tip: press Ctrl-F on Windows or ⌘-F on Mac to search this page.

What this app does

Archival spreadsheets often contain dates entered in many different ways: 5/8/2026, May 8, 2026, 2026-05-08, circa 2026, 1960s, n.d., and so on. The Date Formatter reads each value in a column you choose and rewrites it in a single consistent format. Values it cannot confidently parse are left alone and flagged for review.

You select the file, you select the columns, and the app shows you which rows it was uncertain about.

Quick start

  1. Make a backup copy of your spreadsheet. The app overwrites the original.
  2. Launch the app and pick a conversion mode (Single Date, Date Range, or Dublin Core).
  3. Click Browse and choose your .xlsx or .csv file.
  4. Tick the columns that contain dates.
  5. Click Run. When it finishes, sort by the Check column to review flagged rows.
Before you start: the app saves on top of your original file. Always work from a copy until you are confident with the output.

Launching the app

There are two ways to start the app. Use whichever your IT Admin set up.

Option A — Desktop shortcut (most common)

  1. Find the shortcut. Look on your desktop for date-formatter-gui.bat (or a similarly named icon). If it is not on your desktop, ask your IT Admin where it is.
  2. Double-click it. The app window opens within a few seconds. A black command window may flash open briefly. That is normal.

Option B — Command Prompt

  1. Open Command Prompt. Press Win+R, type cmd, and press Enter.
  2. Run the script. Type the command below and press Enter. 
    py %USERPROFILE%\scripts\date-formatter-gui.py
    Your IT Admin may have configured a different path.

Once the app is open, you will see five stacked sections. From top to bottom:

The five-step workflow

Converting a spreadsheet is always the same five steps. Each section below walks through one step, what to click, and what you should see when it works.

  1. Pick a mode — choose the output style.
  2. Open your file — load the spreadsheet.
  3. Pick date columns — tell the app which columns to convert.
  4. Run it — let the app do its work.
  5. Review flagged rows — check the rows the app was unsure about.
Step 1 of 5

Pick a conversion mode

Three radio buttons sit at the top of the window. Each one produces a different output style. Pick the one that matches the system you are feeding the cleaned data into.

Which mode should I pick?

If you need…Pick this mode
One specific date per row (e.g. event date)Single Date
A date range per row, for ArchivERA importsDate Range (ArchivERA)
Dublin Core or ISO-style date strings cleaned upDublin Core
You are not sureDate Range — it is the safest general-purpose choice

Single Date Conversion — output MM/DD/YYYY

Use this when each row should resolve to a single date.

  • 5/8/2026 becomes 05/08/2026.
  • May 8, 2026 becomes 05/08/2026.
  • If the original value is a range like 01/01/1962 - 12/31/1962, the output is the start date 01/01/1962.
  • Vague values such as circa 1962, 1960s, before 1991, or undated produce an empty result and are flagged for review.

ArchivERA / Date Range Conversion — output MM/DD/YYYY - MM/DD/YYYY

Use this when each row represents a span of time. This mode is designed for ArchivERA imports.

  • 1962 becomes 01/01/1962 - 12/31/1962.
  • Jun 1962 becomes 06/01/1962 - 06/30/1962.
  • 1960s becomes 01/01/1960 - 12/31/1969.
  • 5/8/2026 stays as 05/08/2026.
  • Fuzzy values are preserved and flagged: circa 1962, before 10/15/1991, after 1991, undated.

Dublin Core Conversion

Use this when you need Dublin Core-friendly date output. Like ArchivERA, this mode emits dates as ranges. It accepts a wider variety of inputs than ArchivERA, including ISO and Dublin Core partial formats (2026-05-08, 1962-1965, 1962-06/1965-08, YYYY-MM-DD/YYYY-MM-DD).

Step 2 of 5

Open your file

  1. Click Browse. A standard Windows file picker opens.
  2. Select an .xlsx or .csv file and click Open. If you have .xls, open it in Excel and save as .xlsx first.
  3. Wait for the file to load. Large files take a few seconds. The filename appears in the file box, the Columns section fills with one checkbox per column, and the status bar shows the row count.
Good to know: the app reads every column as text. Leading zeros, codes like 001.001, and folder numbers like 04 are preserved exactly as they appear in the source.
Step 3 of 5

Pick date columns

  1. Tick every column that contains dates. You can pick more than one in a single run (for example both Full Date and Date Checked). Ticked columns process independently, in the order you ticked them.
  2. Leave non-date columns unticked. Anything you do not tick is left exactly as it was.
Not sure which columns to tick? Pick one and run the app on a copy. The Original_ column preserves the raw value so you can always compare.
Step 4 of 5

Run it

Last chance for a backup. When the run finishes, your original file is overwritten with the cleaned version. If you have not made a backup copy yet, do that now.
  1. Close the spreadsheet in Excel if it is open. The app cannot save to a file Excel is holding.
  2. Click Run. The progress bar advances and the status line names the row and column being processed.
  3. Wait for it to finish. Large files with many columns can take a minute or two. The status line reports either "Done. N rows flagged for review" or "Done. N rows processed, no rows flagged."
How to tell it worked: the status line shows a "Done" message and your file now contains Original_ and Check columns next to each date column you ticked.

If "File in use" appears

Excel (or another program) still has the file open. Close it and click Retry.

Step 5 of 5

Review flagged rows

For every column you ticked, the app adds two new columns next to it.

ColumnContents
{your column}The cleaned, formatted date. Replaces the original value in place.
Original_{your column}The raw original value, preserved for reference.
Check {your column}Yes if the app was uncertain about the row. Blank otherwise.

For example, ticking Full Date yields:

... | Full Date | Original_Full Date | Check Full Date | ...

How to review

  1. Open the file in Excel.
  2. Sort by the Check column descending. All flagged rows rise to the top.
  3. Compare each flagged row's Original_ value to the formatted output. Edit the formatted column if you need to fix it.
  4. Save the file.

Why a row gets flagged

A row is flagged when any of these are true:

  • The original value was vague (circa 1962, before 1991).
  • The app could not recognize the format.
  • The original cell contained multiple values separated by semicolons.
  • The result did not match a strict MM/DD/YYYY or MM/DD/YYYY - MM/DD/YYYY shape.
Flagged rows are not errors. They are rows that benefit from a human glance.

Built-in safeguards

Every formatted output respects three rules:

  • Correct day counts per month. February has 28 days (29 in leap years). April, June, September, and November have 30. The rest have 31. The app never emits an invalid date like 02/30/1990 or 04/31/1990.
  • Leap-year awareness. February 29 only appears in years where it is valid. A year is a leap year if it is divisible by 4, except century years which must also be divisible by 400. So 02/29/2000 is valid and 02/29/1900 is not.
  • Chronological order. In any output range start - end, the start date is on or before the end date. If the input had them reversed, the app swaps them before saving.

If the input itself is an invalid date (for example 02/30/1990), the app leaves it alone and flags the row for review rather than guessing what was meant.

Special outputs

These values appear when the input expresses something more nuanced than a plain date or range. They are always flagged.

OutputMeaning
undatedThe original said n.d., no date, not dated, etc.
circa 1962The original said circa 1962, ca. 1962, approx. 1962, etc.
before 01/01/1991The original said before 1991, pre-1991, or ante 1991.
before 10/15/1991The original gave a specific cutoff date with "before".
after 12/31/1991The original said after 1991 or post-1991.
after 10/15/1991The original gave a specific cutoff date with "after".

For year-only inputs, "before" lands on January 1 of that year and "after" lands on December 31. This is a tight bound: the named year is excluded from the range.

What gets cleaned vs. what stays the same

Always cleaned, regardless of mode

The following archival ID columns have their leading zeros restored on every save.

Column header (any variant)Format
RG, Record Group NumberExactly 4 digits. 200 becomes 0200. 9200 stays 9200.
SG, SubGr, SubGroup, Subgroup NumberExactly 3 characters. 22 becomes 022. W22 stays W22. Single-letter prefix is preserved.
Series, Series NumberSame as SG.
SubSeries NumberSame as SG.

Left alone

Every other column in your spreadsheet, including Folder Number, Sequential Box Number, Title, Description, and Container Barcode, is preserved exactly as it appeared in the source. Leading zeros are kept and codes like 001.001 are not converted to numbers.

If something goes wrong

Common problems and what to try first.

SymptomWhat to do
Error dialog appears Click OK. The app writes details to %TEMP%\date-formatter.log. Open it in Notepad and share with your IT Admin.
"File in use" prompt Close the spreadsheet in Excel (or whatever has it open), then click Retry.
The app does not open Python may not be installed, or the script path may have changed. Contact your IT Admin.
A row was flagged that should not have been Check the Original_ column for the raw value the app saw. Edit the formatted column directly, or clean up the source value and re-run.
Nothing was flagged but a row looks wrong Open the row, compare the formatted value to Original_, and edit the formatted column if needed. The Check column is a hint, not a guarantee.
Progress bar is stuck Wait a minute — large files can pause while saving. If still stuck, close the app, check %TEMP%\date-formatter.log, and re-run on a copy of the file.

Tips

  • Make a backup copy of your spreadsheet before running. The app overwrites your file in place.
  • Run on one column at a time until you are comfortable with the output, then batch larger jobs.
  • After a run, sort by the Check column to bring flagged rows to the top.
  • The app handles .xlsx and .csv. Save older .xls files as .xlsx first.
  • Light mode and dark mode are both supported. Your preference is saved.

Need help

Contact your IT Admin or whoever provided this app. Include the log file from %TEMP%\date-formatter.log if you encountered a specific error.
