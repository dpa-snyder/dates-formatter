Guiding rule: No behavioral changes to date logic at any phase. All processing stays in date_engine.py. Flask routes are thin wrappers only. The existing CTk GUI stays runnable until W-505 (smoke test) is confirmed.

PHASE 1

Logic Extraction

Isolate all date parsing and formatting into a pure Python module. No UI imports. Tests pass before and after. This phase is a prerequisite for everything else — complete it in isolation before writing any Flask code.

PHASE 2

Flask Backend

HTTP API layer that wraps date_engine. Routes validate inputs, call the engine, and return results. No processing logic lives in routes.

Low risk 7 tasks
▾
ID Task Notes
W-201
Scaffold web/app.py
Flask app skeleton. Host: 127.0.0.1 only (no LAN exposure). Debug off in production. Blueprint structure if routes grow beyond one file.
Add flask to requirements.txt. No other Flask extensions needed.
W-202
POST /api/load — file upload
Accept multipart/form-data with xlsx or csv. Save to tempfile.mkdtemp(). Return JSON: {columns, row_count, token}. Token is a UUID mapped server-side to the temp path.
Reject non-xlsx/csv with 400. Max file size: configurable constant, default 50 MB. Token dict lives in module-level dict (single-user local app, no concurrency concern).
W-203
POST /api/process — run conversion
Accept: {token, column, mode, output_mode}. Look up temp file by token. Call engine. Stream result file back as download. Clean temp files after response.
Modes: single / ae / dublin. Output modes: overwrite / copy. Return Content-Disposition: attachment. Orphaned temp files cleaned on next server start.
W-204
GET /api/settings
Return current dates-formatter-settings.json as JSON. Same schema as existing sidecar file.
Settings file path: same location as existing sidecar (os.path.dirname(os.path.abspath(__file__))).
W-205
POST /api/settings
Merge incoming JSON with defaults, write updated settings file. Return updated settings.
Validate keys against known schema — reject unknown keys with 400 to prevent settings corruption.
W-206
GET /manual — serve user manual
Serve prod/user-manual.html via send_from_directory. Opens in new browser tab from frontend link.
Path resolves relative to app.py. No template rendering — static serve only.
W-207
Consistent JSON error handling on all routes
All error responses: {"error": "human-readable message"} with appropriate HTTP status. No Flask HTML error pages escape to the browser.
Register @app.errorhandler for 400, 404, 500. Errors also written to existing log path (date-formatter.log).

Temp file strategy: Each upload gets a UUID. Server holds {uuid: tmp_path} in memory. File deleted after /api/process streams. Orphaned files (server crash, no process call) cleaned on next startup by scanning tempdir for known prefix.

PHASE 3

Frontend

Single-page app at web/templates/index.html. Vanilla JS, no framework, no bundler. Clean native-feeling design — not a data tool aesthetic. Fetches against the Flask API.

Low risk 8 tasks
▾
ID Task Notes
W-301
Design system + base template
web/templates/index.html with web/static/style.css. CSS custom properties for color and spacing. Dark/light mode via data-theme attribute. Matches existing palette: #235194 blue, #e5842a orange. IBM Plex fonts.
Single HTML file served by Flask template engine. CSS and JS as separate static files for caching.
W-302
File upload zone
Drag-and-drop target + click-to-browse fallback (<input type="file">). On file select: POST to /api/load. Show filename, row count, column count in a file card on success.
Accept .xlsx, .csv only. Show loading state during upload. File card replaces drop zone after successful load.
W-303
Column picker
Dropdown/select populated from /api/load response column list. Shows column names. Required before Run is enabled.
If date_columns hint present in response (W-503), auto-select first suggested column and visually flag suggested columns.
W-304
Mode selector + output mode toggle
Three mode options: Single Date / Date Range (ArchivERA) / Dublin Core. Per-mode description text and output format example below selector. Output mode toggle: Overwrite in place / Save as copy.
Mode defaults from settings. Output mode example updates dynamically when mode changes.
W-305
Run → spinner → download
Run button POSTs {token, column, mode, output_mode} to /api/process. Response is a file blob → trigger browser download. Status text shows rows processed and flagged count after completion.
Spinner state while request is in flight. Disable Run during processing. On error: show inline error message, re-enable Run.
W-306
Settings panel
Accessible via gear icon (top-right). Slides in as a side panel or modal. Contains: theme toggle (dark/light/system), default output mode. Changes POST to /api/settings immediately (no explicit Save button).
Load initial values from GET /api/settings on page load.
W-307
Inline error states
File format errors, bad column, engine errors. Displayed inline near the relevant control — not alert dialogs. Dismissable.
Network errors (Flask not running): show a distinct "server not responding" message with a reload prompt.
W-308
User manual link
Button or link that opens /manual in a new tab. Positioned consistently with existing CTk "View User Manual" button.
No JS required — plain <a href="/manual" target="_blank">.

JS strategy: Fetch API for all calls. No jQuery. No bundler. Single web/static/app.js. Keep state in a plain JS object. This is a utility, not a dashboard — avoid unnecessary abstraction.

PHASE 4

Launcher

Single entry point that finds a free port, starts Flask, opens the browser, and handles shutdown cleanly on both platforms.

Low risk 4 tasks
▾
ID Task Notes
W-401
Create prod/launch.py
Free port via socket.bind(('', 0)). Start Flask in a daemon thread. Poll GET http://127.0.0.1:{port}/ until ready (max ~5s). Open http://127.0.0.1:{port} in default browser. Block on KeyboardInterrupt.
Print port to stdout on launch. Flask thread is daemon so it dies when main thread exits. No external dependencies beyond stdlib + flask.
W-402
Windows launcher: date-formatter-web.bat
Calls py prod\launch.py. Keep existing date-formatter-gui.bat until W-506 (CTk retirement).
Batch file lives in prod/. Can be copied to Desktop. If py not found, fallback to python.
W-403
Mac launcher: date-formatter-web.command
Double-clickable shell script on Mac. #!/bin/bash. cd to script dir, call python3 launch.py. Requires chmod +x after copy.
Document chmod +x step in README. Mac Gatekeeper may prompt on first open — document that too.
W-404
Graceful shutdown
Ctrl+C in terminal: clean temp dir, exit. Browser window close: best-effort via window.addEventListener('beforeunload', ...) → navigator.sendBeacon('/api/shutdown'). Flask route sets a flag to stop the server loop.
Shutdown is best-effort. If browser close doesn't fire, process exits when terminal is closed or Ctrl+C is used. Temp dir is always cleaned on next startup.

STRETCH B

PyWebView — Native Window

Wrap the Flask app in a native webview window. Same HTML/CSS/JS frontend, same Flask backend — but no browser chrome, no URL bar, no tabs. Feels like a standalone desktop app.

Stretch Medium risk 5 tasks
▾
ID Task Notes
W-SB1
Add pywebview>=4.4 to requirements
Verify install on both platforms. Windows uses Edge WebView2 (present on Win11 + modern Win10 by default). Mac uses WebKit.
Do not add to base requirements.txt until Stretch B is confirmed in scope. Keep in a separate requirements-native.txt until then.
W-SB2
Create prod/launch_native.py
Start Flask in background thread (same port-pick logic as launch.py). Open pywebview window at http://127.0.0.1:{port}. Window title: "Dates Formatter". webview.start() blocks until window closed → Flask thread exits with it.
Minimum window size: 900×650. Resizable. No frameless=True — keep OS chrome for accessibility (minimize, resize, close buttons).
W-SB3
Native launchers
Windows: date-formatter-native.bat. Mac: date-formatter-native.command (chmod +x required). Both call launch_native.py.
Keep browser-based launchers (W-402, W-403) as fallback.
W-SB4
Platform rendering test
Windows: confirm Edge WebView2 renders layout correctly, drag-and-drop works, download triggers correctly. Mac: confirm WebKit renders identically. Flag any CSS or JS differences.
File download inside pywebview requires care — webview.windows[0].evaluate_js may need to trigger the download differently than a standard browser.
W-SB5
Fallback documentation
If Edge WebView2 is not installed on a Windows machine, document the fallback: run date-formatter-web.bat (browser version) instead. Add a one-liner error message in launch_native.py that catches the missing WebView2 error and prints the fallback instruction.
Edge WebView2 installer: available free from Microsoft. May be worth bundling the bootstrapper URL in the error message.

What Does Not Change

Date Parsing

All patterns, edge cases, error detection. Zero logic changes across all phases.

Output Columns

{col}, Original_{col}, Check {col} structure unchanged.

Conversion Modes

Single / AE Range / Dublin Core behavior and output format unchanged.

Output File Format

xlsx and csv output. Same file structure as today.

Settings Schema

Existing keys: theme, geometry, output_mode. Web adds new keys alongside, never removes.

User Manual

user-manual.html content unchanged. Served at /manual instead of opened via webbrowser.

Test Suite

./run-tests.sh must pass at the end of Phase 1 and remain passing at every subsequent phase.

CTk GUI

Stays functional until W-505 smoke test passes and client sign-off received (W-506).
